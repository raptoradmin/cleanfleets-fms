I met with Sean last night and as a result, there were several modifications identified that need to be made to the CleanFleets system. One of those changes was the ability for the system to accommodate the need to have a record of annual miles and hours for the vehicle and/or engines.
 
After spending quite some time last night contemplating what changes I need to make to do this, I find this to be a more challenging task than I first anticipated. The reason for this is because an engine is inventoried separately from the chassis.
 
I first considered just moving the MILEAGE FIELD and HOURS FIELD from Vehicle to Engine, however, miles do count toward the vehicle chassis so mileage really needs to stay with Vehicle. This begs the question whether or not mileage applies to the engine or do only hours apply to engine?
 
In any event, this will require adding two new tables to the system:
 
CF_Engines_Hours
CF_Vehicles_Mileage
 
Unless told otherwise, I plan to include both the mileage and hours in both tables just to cover any eventuality. We need to do this in case of that rare event that one engine is moved from one vehicle to another. The vehicle it was removed FROM needs to still be able to show Mileage and the Engines need to be able to have a record of Hours. CARB may also require the mileage be applied to the engine as well as hours thus, just keep both miles and hours associated with both the chassis and the engine.
 
The other major item is the use of a hand held device to inventory fleets. To enable the current data system to manage records from the hand held devices, I will also be adding a new field to each of the following tables:

CF_Vehicles

IDVehiclesHH

CF_Engines

IDEnginesHH

CF_DECs

IDDECsHH

CF_Images

IDImagesHH

This field will be populated by the ID field generated by the Hand Held system ID in order to add records back once a vehicle has been inventoried. What I recommend is that each installation of the hand held software have a unique identifier so we can distinguish one from another, then combine that identifier with an inserted record ID so that the field contains the hand held identifier AND the incremented ID.
 
My recommendation to begin the process of inventory by hand held would be that the user must first create an Account, Terminal, and a Fleet. Once the Fleet is created, I can include a button to send the Fleet ID to the hand held process.

Here is what we will need back from the hand held:

Either

One to One records for Vehicles, Engines, DECs, and Images

Or

One to One records for Engines not in a vehicle with a DECS attached, and Images

Or

Single records for

Vehicles without engines and Images

Engines not attached to a vehicle and Images

DECs not attached to an engine and Images

In ALL cases the Fleet ID should be added to each Vehicle, Engine, and/or DECS record because we have to keep track of what fleet each one belongs to because each one is inventoried separately.
 
These are major changes to the current application. It is going to take me some time to redo the logic for the two new tables and how to manage the records added for mileage and hours.
 
Once I have reconfigured the database by adding the new tables and fields, I will create a new table dictionary and send it out.
